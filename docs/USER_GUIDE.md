# 用户操作手册

> 项目：错题批改助手
> 面向：家长（最终使用者）
> 适用环境：手机浏览器 / 桌面浏览器

---

## 1. 快速开始

### 1.1 本地预览（开发）
```bash
cd ~/homework-checker
npm run dev
```
浏览器打开 http://localhost:3000

> 若手机也想访问，需在同一 Wi-Fi 下用电脑局域网 IP 替换 localhost（开发服务已绑定 0.0.0.0）。

### 1.2 首次可用前提
应用采用了"零配置可启动、装好密钥即批改"的设计。功能可用性取决于配置：

| 配置项 | 未配置时 | 配置后 |
|--------|---------|--------|
| Claude API Key | 批改返回明确提示"未配置" | ✅ 可真实批改 |
| Supabase | 错题本显示空、无法存档 | ✅ 可保存回看 |
| Google OAuth | 登录按钮可点但跳转失败 | ✅ 可登录 |

配置步骤见 `DEPLOYMENT.md`。

## 2. 首页用法（拍题批改）

### 步骤
1. **选中图片**：点击页面上方虚线区「📷 点击选择题目图片」。
   - 手机上会弹出"拍照/相册"选择。
   - 桌面上可选本地图片。
2. **预览确认**：图片出现在上传区，下方「开始批改」按钮由灰变亮。
3. **开始批改**：点击「✨ 开始批改」。
   - 按钮变为「⏳ 批改中...Ns」，转圈动画 + 秒数计时。
   - 通常需 5~15 秒。
4. **查看结果**：批改结果卡片出现，包含：
   - 错误之处
   - 正确解答
   - 错因分析
   - 涉及知识点
   - 若题目全对，会提示"全部正确"。
5. **存入错题本**：点击底部「💾 存入错题本」，按钮变为「✓ 已存入错题本」。

### 异常情况
| 现象 | 含义 | 处理 |
|------|------|------|
| 红框 ⚠️ 未配置 Claude API Key | 未填密钥 | 按 DEPLOYMENT.md 配置并重启 |
| 红框 ⚠️ 批改超时，请重试 | AI 响应超 30s | 重新点开始批改 |
| 红框 ⚠️ 图片过大 | 超过 10MB | 重新拍清晰但更小的图 |
| 红框 ⚠️ 网络错误 | 断网 | 检查网络重试 |
| 「存入」后变灰但仍可看 | 已存库 | 无需重复点 |

## 3. 错题本用法

### 进入
首页点「📋 错题本」，或直接访问 `/history`。

### 界面状态
- **加载中**：几秒内显示灰色骨架条（占位动画）。
- **空**：提示"暂无错题记录"+「去添加第一道错题」按钮，点回首页。
- **有数据**：每条卡片显示学科标签 + 时间 + 题目摘要。
- **错误**：显示 ⚠️ 错误文字 +「重试」按钮。

### 查看错因
点击任意一条，下方展开「错因分析」详情；再次点击收起。

## 4. 登录

- 首页点「🔗 Google登录」，跳转 Google 授权。
- 授权后回到应用；登录主要用于后续多设备数据同步。
- 当前阶段批改功能不强制登录。

## 5. 常见问题（FAQ）

**Q：为什么我点的批改一直转圈？**
A：AI 通常 5~15s。若超过 30s 会自动提示超时；请重试或检查网络。

**Q：拍的照片很清楚为什么 AI 没识别全？**
A：手写题识别受光照、字迹影响。尽量光线均匀、字迹端正、只拍题目区域。

**Q：错题本点了"存入"却没有？**
A：需已配置 Supabase。未配置时存档不可用，会有相应提示。

**Q：在手机上要装 App 吗？**
A：不需要。这是网页应用，浏览器打开即可；可"添加到主屏幕"获得类 App 体验。
