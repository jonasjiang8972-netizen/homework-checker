# 用户操作手册

> 项目：作业小帮手
> 面向：初中生（主要使用者）/ 家长（辅助配置）
> 适用环境：手机浏览器 / 桌面浏览器
> 更新日期：2026-06-30

---

## 1. 快速开始

### 1.1 本地预览（开发）
```bash
cd ~/homework-checker
npm run dev
```
浏览器打开 http://localhost:3000

> 若手机也想访问，需在同一 Wi-Fi 下用电脑局域网 IP 替换 localhost（开发服务已绑定 0.0.0.0）。

### 1.2 首次可用前提
应用采用了"零配置可启动、装好密钥即批改"的设计。功能可用性取决于配置：

| 配置项 | 未配置时 | 配置后 |
|--------|---------|--------|
| Claude API Key | 批改返回明确提示"未配置" | ✅ 可真实批改 |
| Supabase | 错题本显示空、无法存档 | ✅ 可保存回看 |
| Google OAuth | 登录按钮可点但跳转失败 | ✅ 可登录 |

配置步骤见 `DEPLOYMENT.md`。

## 2. 首页用法（拍题批改）

### 步骤
1. **选中图片**：点击页面上方虚线区「📷 点击选择题目图片」。
   - 手机上会弹出"拍照/相册"选择。
   - 桌面上可选本地图片。
2. **预览确认**：图片出现在上传区，下方「帮我看看」按钮由灰变亮。
3. **开始批改**：点击「帮我看看」。
   - 按钮变为 spinner + 秒数计时。
   - 通常需 5~15 秒。
4. **查看结果**：批改结果卡片出现，包含：
   - 对错判断
   - 订正类型 + 知识点
   - 💡 想一想（引导提示，不给答案）
5. **如果还是不会**：点击「还是不太懂，给我看看答案 →」
   - 展开完整解题过程和错因分析
6. **存入错题本**：点击「记下来 📖」

### 两段式批改说明（v2.8）
- 批改后**先看到的是引导提示**，而不是完整答案
- 这样设计是为了帮你**自己发现错误**，而不是直接抄答案
- 如果自己想明白了就不需要点「给我看看答案」
- 点了「给我看看答案」之后才会显示完整解答

### 隐私模式（v2.8）
- 默认是「学生模式」，在成长日记里**只能看到汇总数据**（做了多少题、正确率、薄弱知识点）
- 家长可以在设置页切换到「家长模式」，查看每一道题的详情
- 这个设计是为了**保护你的学习隐私**——学习数据是你自己的

### 异常情况
| 现象 | 含义 | 处理 |
|------|------|------|
| 红框 ⚠️ 未配置 Claude API Key | 未填密钥 | 按 DEPLOYMENT.md 配置并重启 |
| 红框 ⚠️ 批改超时，请重试 | AI 响应超 30s | 重新点开始批改 |
| 红框 ⚠️ 图片过大 | 超过 10MB | 重新拍清晰但更小的图 |
| 红框 ⚠️ 网络错误 | 断网 | 检查网络重试 |
| 「存入」后变灰但仍可看 | 已存库 | 无需重复点 |

## 3. 成长日记（错题本）

### 进入
首页点「成长日记」，或直接访问 `/history`。

### 界面状态
- **加载中**：几秒内显示灰色骨架条（占位动画）。
- **空**：提示"还没有做过题呢"+「去试试吧」按钮，点回首页。
- **有数据（学生模式）**：显示做题总数、正确率、涉及知识点数、薄弱知识点列表、最近记录简表。
- **有数据（家长模式）**：每条卡片显示学科标签 + 时间 + 题目摘要，可展开查看错因。

### 查看错因（仅家长模式）
点击任意一条，下方展开「错因分析」详情；再次点击收起。

## 4. 登录
- 设置页输入邮箱地址，点击「发送验证码」。
- 查收邮件获取 6 位验证码，输入后点击「登录」。
- 登录后可以保存自己的 API Key 和学习数据。
- 如果使用家长模式查看详情，需要先登录。

## 5. 常见问题（FAQ）

**Q：为什么我点的批改一直转圈？**
A：AI 通常 5~15s。若超过 30s 会自动提示超时；请重试或检查网络。

**Q：拍的照片很清楚为什么 AI 没识别全？**
A：手写题识别受光照、字迹影响。尽量光线均匀、字迹端正、只拍题目区域。

**Q：错题本点了"存入"却没有？**
A：需已配置 Supabase。未配置时存档不可用，会有相应提示。

**Q：在手机上要装 App 吗？**
A：不需要。这是网页应用，浏览器打开即可；可"添加到主屏幕"获得类 App 体验。
