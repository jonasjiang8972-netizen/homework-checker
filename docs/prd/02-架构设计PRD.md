## 1. 架构目标与约束

### 1.1 商用化架构目标
本系统定位为生产环境使用的企业级工具，架构设计需满足以下商用标准：

| 维度 | 商用标准 | 对应措施 |
|---|---|---|
| 高可用 | 核心服务无单点故障，SLA目标≥99.5% | 多副本部署+负载均衡 |
| 可扩展 | 支持用户量/调用量增长，模型厂商可插拔扩展 | 微服务化+模型适配层 |
| 安全合规 | 数据出域可控、敏感信息可脱敏、操作可审计 | 独立脱敏引擎+审计日志 |
| 数据可靠 | 报告数据不丢失，支持备份恢复 | 主从复制+定期备份 |
| 可观测 | 系统自身需要被监控 | 接入现有观测云平台 |
| 成本可控 | AI调用成本可统计、可预警、可限流 | AI网关配额管理 |

### 1.2 关键约束
- 依赖第三方AI大模型API（外部网络调用），需考虑网络出口安全与调用稳定性
- 上传材料含截图/日志，可能包含敏感信息，需满足数据安全合规要求
- 需与现有观测云平台共存，本系统作为独立应用不侵入观测云原有架构

## 2. 总体架构

### 2.1 架构分层

| 层级 | 组件 | 职责 |
|---|---|---|
| 接入层 | Web前端(SPA) / Nginx反向代理 / API网关 | 流量入口、限流、鉴权 |
| 应用服务层 | 材料接收服务、分析任务服务、报告服务、用户权限服务 | 核心业务逻辑 |
| 中间能力层 | 任务队列/MQ、AI模型适配网关、脱敏处理引擎、对象存储 | 异步处理、模型调度、安全 |
| 数据层 | 关系型数据库 + 对象存储 + 全文检索引擎(可选) | 数据持久化 |
| 基础设施层 | 容器编排(K8s) / 日志采集 / 监控告警 / CI-CD | 运行环境 |

### 2.2 核心设计思想
1. **异步解耦**：材料提交与AI分析解耦，通过消息队列异步处理
2. **模型可插拔**：AI大模型能力通过统一的"模型适配网关"接入
3. **数据与凭证分离**：原始材料存对象存储，结构化元数据存关系型数据库
4. **安全前置**：脱敏处理作为独立中间件，在数据出域之前统一拦截处理

## 3. 技术选型建议

| 层级 | 建议选型 | 说明 |
|---|---|---|
| 前端 | React/Vue + TypeScript | 组件化开发 |
| 后端语言/框架 | Java(Spring Boot) 或 Go(Gin/Kratos) | 视团队既有技术栈选择 |
| 任务队列 | Kafka 或 RabbitMQ / Redis Stream | 中小规模用RabbitMQ/Redis Stream |
| 关系型数据库 | MySQL 8.0 / PostgreSQL | 建议主从/多副本部署 |
| 对象存储 | 自建MinIO 或 云厂商OSS/S3 | 存放截图、日志原文件 |
| 全文检索（可选） | Elasticsearch | 历史记录关键字检索 |
| 容器编排 | Kubernetes | 商用级要求 |
| 网关 | Nginx / APISIX / Kong | 统一入口、限流、鉴权 |
| 配置中心（可选） | Nacos / Apollo | 动态配置管理 |

## 4. 高可用架构设计

| 组件 | 高可用方案 |
|---|---|
| Web/API服务 | 无状态化设计，K8s多副本+HPA自动扩缩容 |
| 数据库 | 主从复制/多可用区部署，主库故障自动切换 |
| 消息队列 | 集群部署，避免任务丢失 |
| 对象存储 | 多副本/纠删码存储方案 |
| AI模型调用 | 多模型/多Endpoint容灾，主模型不可用自动降级 |
| 整体容灾 | 同城双活或主备部署 |

## 5. AI模型适配网关设计（核心模块）

### 5.1 统一接口抽象
```
IModelAdapter {
  analyze(materials, prompt_template) → StructuredRCAResult
}
```

### 5.2 关键能力

| 能力 | 说明 |
|---|---|
| 路由与降级 | 主模型不可用自动降级到备用模型 |
| 限流与熔断 | 防止突发流量打满第三方API配额 |
| 结构化输出校验 | 强制按预定义JSON Schema返回 |
| 调用日志与追溯 | 记录请求/响应、耗时、Token消耗 |
| Prompt版本管理 | 模板变更可追溯版本 |
| 引用定位输出 | 根因结论标注依据来源（哪张图/哪几行日志） |

### 5.3 成本与配额管理
- 按模型/按部门/按用户维度统计Token消耗
- 支持配置调用配额上限

## 6. 数据架构设计

### 6.1 数据分类与存储策略

| 数据类型 | 存储位置 |
|---|---|
| 用户/权限数据 | 关系型数据库 |
| 分析任务元数据 | 关系型数据库 |
| 报告结构化数据 | 关系型数据库（JSON字段或独立表） |
| 原始材料（截图/日志文件） | 对象存储 |
| AI调用日志（脱敏后） | 关系型数据库/日志系统 |
| 操作审计日志 | 独立日志表或日志系统 |

### 6.2 核心数据模型 — rca_report（报告主表）

| 字段 | 说明 |
|---|---|
| id | 主键 |
| task_id | 关联分析任务 |
| alert_name | 告警名称 |
| service_name | 所属服务 |
| alert_level | 告警级别 |
| occur_time | 发生时间 |
| detect_time | 检测时间 |
| recover_time | 恢复时间 |
| status | 状态 |
| impact_scope | 影响范围 |
| impact_severity | 影响严重程度 |
| root_causes | 根因分析（JSON数组） |
| evidence_refs | 分析依据（JSON） |
| recommendations | 处置建议（JSON数组） |
| model_used | 实际调用的AI模型 |
| feedback_status | 人工反馈状态 |
| feedback_note | 人工反馈备注 |
| created_by / created_at | 创建人/时间 |

### 6.3 数据生命周期与留存策略
- 报告数据保留24个月
- 原始材料保留12个月后归档/清理
- 支持软删除+回收站机制

## 7. 安全架构设计

### 7.1 身份认证与权限
- 统一接入企业SSO（LDAP/OAuth2/SAML）
- 基于RBAC的细粒度权限控制

### 7.2 传输与存储安全
- 全链路HTTPS/TLS加密
- 对象存储静态加密（SSE）
- 数据库敏感字段加密存储

### 7.3 数据出域安全（重点）
```
原始材料 → 脱敏引擎（IP/密钥/手机号/身份证等敏感模式识别） → 脱敏后内容 → AI模型API
```
- 脱敏规则可配置
- 提供脱敏效果预览
- 记录完整的数据出域审计日志

### 7.4 审计与合规
- 全量操作日志留存
- 定期安全评估（渗透测试、SCA）

---

## 8. 部署架构

### 8.1 部署拓扑
- 负载均衡 → 多可用区K8s集群 → 数据库主从 + 消息队列集群 + 对象存储

### 8.2 环境规划
- 至少三套环境：Dev / Staging / Prod
- CI-CD流水线：代码提交 → 测试 → 镜像构建 → 预发布验证 → 灰度/全量发布

### 8.3 灾备与备份
- 数据库每日全量备份+实时增量备份
- 对象存储开启版本控制
- 定期演练故障恢复流程
