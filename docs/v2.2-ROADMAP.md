# v2.2 优化路线图

## 🎯 优化目标

为 v2.1 所做的基础准备，整体提升产品性能、扩展性和用户体验。

## 🔧 核心优化方向

### 1. 导入更多模型支持

#### 目标
实现多种 AI 模型切换功能，提升对不同类型题目的处理准确度。

#### 功能要求
- 支持 Claude 3.5 Sonnet / Haiku / Opus 等多个版本
- 后端配置模型切换
- 前端支持模型类型选择

#### 当前实现策略
使用 `ANTHROPIC_MODEL` 环境变量控制模型：
```ts
const MODEL_MAP: Record<string, string> = {
  'claude-3-haiku-20240307': 'claude-3-haiku-20240307',
  'claude-3-5-haiku-latest': 'claude-3-5-haiku-latest',
  'claude-3-5-sonnet-latest': 'claude-3-5-sonnet-latest',
  'claude-3-opus-20240229': 'claude-3-opus-20240229',
};

const DEFAULT_MODEL = 'claude-3-5-sonnet-latest';
```

### 2. 添加移动端适配

#### 目标
提升在移动设备上的使用体验，确保响应式布局和交互优化。

#### 功能要求
- 移动端友好的界面适配
- 触控优化的交互组件
- 响应式图片上传界面

#### 当前实现策略
- 在布局定义中使用 `viewport` 元标签
- 视图区域采用适合移动设备的百分比和弹性布局

### 3. 开发配套 AI 调试工具

#### 目标
为开发者和高级用户提供调试和优化 AI 批改结果的能力。

#### 功能要求
- 模型输出预览面板
- 批改结果的可切换版本展示
- OCR 文本和图像对比视图

### 4. 增加数据回收策略

#### 目标
优化大数据存储，支持分片和过期数据自动清理。

#### 功能要求
- 定期删除历史记录
- 设置数据保留周期
- 批量删除功能

#### 当前实现策略
- 为 `batch_grading_data` 表设计了时间索引
- 未来将引入数据分片和定期清理逻辑
