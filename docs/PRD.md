## 1. 产品概述

### 1.1 一句话定义
一款面向小学生家长（用户本人）的移动端 Web 应用：**拍照上传错题，AI 立即批改并拆解错因，自动归入错题本供后续复习**。

### 1.2 产品意图与背景
用户为初次编程的学习者，为其女儿开发此应用。核心痛点：
- 孩子做错的题目，家长批改费时，且不一定能讲清"错在哪、为什么错"；
- 错题散落各处，没有沉淀，无法针对性复习；
- 孩子不会的题目无人讲解，缺乏即时反馈。

本产品用 AI 摄像批改替代人工批改，用错题本沉淀替代纸质收集，目标是一个家长 3 次点击内就能完成「拍题 → 看批改 → 存档」的闭环。

### 1.3 产品目标
| 目标 | 衡量指标 |
|------|----------|
| 批改准确可用 | AI 能识别对错并给出正确解法与错因 |
| 流程极简 | 从打开到看到批改结果 ≤ 3 次点击 |
| 错题可沉淀 | 题目+错因+时间可保存可回看 |
| 体验流畅 | 移动端适配、有进度反馈、不因配置缺失崩溃 |

## 2. 用户画像

### 2.1 主要用户：家长（产品所有者）
- 角色：拍照、发起批改、判断结果、存档管理
- 设备：MacBook 开发 + 手机日常使用
- 技术水平：非技术背景，依赖 AI 辅助编程
- 核心诉求：少操作、快出结果、看得懂错因

### 2.2 最终受益者：女儿（学生）
- 角色：被批改对象、潜在复习功能使用者
- 场景：做完作业 → 家长拍照批改 → 针对错题讲解或复习

### 3.2 核心 use case
**UC-1 拍照批改（主流程）**
1. 家长打开 App（首页）
2. 点击拍照区，选/拍一张题目图
3. 预览确认无误，点击「开始批改」
4. 等待 5-15s（看到计时与 spinner）
5. 查看「批改结果」卡片
6. 点击「存入错题本」

**UC-2 回看错题**
1. 首页点「错题本」
2. 浏览列表，点击某题展开错因分析

## 4. 体验设计意图

### 4.1 交互层级
遵循「渐进展示」原则，降低认知负担：
- **未选图**：批改按钮禁用，不引导无效操作
- **已选图**：按钮激活，预览可见
- **批改中**：spinner + 秒数，消除"卡死"焦虑
- **出错时**：红框明确文字（非技术报错），可重试
- **空状态**：错题本为空时引导"去添加第一道错题"

### 4.2 视觉语言
| 元素 | 取值 | 意图 |
|------|------|------|
| 主色渐变 | `#667eea → #764ba2` | 专业且温暖，区别于纯工具感 |
| 卡片白底圆角 24px | 内容容器 | 聚焦信息，移动端柔和 |
| Emoji 图标 | 📚📷✨⚠️✓ | 零依赖负担，直观亲和 |
| 骨架屏 shimmer | 加载占位 | 优于纯文字"加载中" |
| 红色错误框 | 失败提示 | 即时可识别异常 |

### 4.3 移动优先
- max-width 480px 居中，桌面端不撑满
- 上传区触控 hit area ≥ 44px
- viewport 锁定缩放，防止双击放大误操作

## 6. v2.0 学习闭环系统需求

### 6.1 升级定位
v1.x 是「批改工具」——拍题→批改→存错题，单次闭环。
v2.0 升级为「自适应学习助理」——数据循环驱动下一轮计划，每次做题都在喂养更精准的学习安排。

### 6.2 六模块闭环
1. **批改**（已有，改造为结构化输出）
2. **打标签**：自动提取知识点名称 + 错误类型
3. **掌握度计算**：每知识点 0-100 分，正确率加权
4. **生成计划**：AI 针对薄弱点拆解提升步骤
5. **测验验收**：同知识点出新题，检验掌握
6. **反馈更新**：测验结果回写掌握度，循环

### 6.3 v2.0 新增功能清单
| 功能 | 优先级 | 状态 |
|------|--------|------|
| 批改结构化输出（JSON：对错/知识点/错型） | P0 | ✅ 已完成 |
| 错题入库携带标签 | P0 | ✅ 已完成 |
| 分块结果卡片展示 | P0 | ✅ 已完成 |

| 同知识点测验出题 | P3 | ✅ 已完成 |
| 测验结果回写掌握度 | P3 | ✅ 已完成 |

### 6.4 数据契约（前后端约定）
批改接口返回结构化 JSON：
```json
{
  "is_correct": false,
  "error_type": "计算失误",
  "knowledge_point": "两位数进位加法",
  "error_spot": "...",
  "correct_solution": "...",
  "analysis": "...",
  "knowledge_tags": ["进位加法", "竖式计算"]
}
```
所有下游功能（热力图/计划/测验）建立在此数据之上。
